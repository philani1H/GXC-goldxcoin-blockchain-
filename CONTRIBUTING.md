# Contributing to GXC-core

Thank you for your interest in improving the GXC Blockchain! We welcome contributions from the community to help make GXC more secure, efficient, and feature-rich.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please open an issue on GitHub with:
- A clear description of the problem.
- Steps to reproduce the issue.
- Your environment details (OS, compiler version).
- Any relevant logs or error messages.

### 2. Suggesting Enhancements
We are always looking for ways to improve GXC. If you have an idea:
- Open an issue to discuss the proposal.
- Explain the use case and the benefit to the network.

### 3. Code Contributions
- Fork the repository.
- Create a new branch for your feature or fix.
- Ensure your code follows the existing style (C++17).
- Write tests for new functionality.
- Submit a Pull Request with a detailed description of your changes.

## Development Guidelines
- **Language**: C++17
- **Build System**: CMake
- **Dependencies**: OpenSSL, LevelDB, Snappy
- **Testing**: Use the existing test suite in the `tests/` directory to validate changes.

## Code of Conduct
Please be respectful and professional in all interactions within the GXC community.
